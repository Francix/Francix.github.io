音乐生成模型综述

符尧 francis_yao@pku.edu.cn

北大信科/ 头条AI实验室

FRI 23TH MAR 2018

 

因为音乐生成的各个模型非常大地依赖于音乐的表示方法，而音乐的表示方法多样，因此讲这个综述我觉得应该从各种不同表示的角度来讲。

 

音乐的原始表示

所谓的原始表示就是音乐的原始文件，先讨论midi

  • 单音轨midi：这种midi 支持和弦，但某些模型不支持和弦
  • 多音轨midi：一条主音轨，一条副音轨，一条鼓点音轨。大部分模型不支持多音轨

在处理了midi文件之后，有One hot，Pianoroll，Events 三种基本表示

 

One-hot 表示

这种表示也叫做一个Melody，不支持和弦，在一个 timestep 时只能有一个键被按下。这样的表示可以直接用 RNN-softmax 来建模。在 Magenta Melody RNN 1 实现了这个模型。这种表示过于 trivial。并且很多数据本身就是带和弦的

 

pianoroll表示

这种表示支持和弦，在同一个timestep 时可以有多个键被按下。所谓的pianoroll 就是指一个向量序列，在每个timestep上，如果某个特定的键被按下，那么其对应位置encoding 为1，否则为0 。这种pianoroll 支持和弦。

可以看出在给定 timestep 时是一个多峰分布，不可以用 softmax 建模。我们曾尝试用对每一维用 sigmoid 来建模，但是这种建模方法没有考虑维度之间的相互依赖，所以效果很差。因此需要用多峰模型。

多峰模型一个经典文章是RNN-RBM2，为Bengio 组 ICML 12的文章 . 这篇文章使用了 RBM 来对多峰模型建模，用RNN 来对时间序列建模

同时这篇文章中提出的另一个模型是 RNN-NADE，就是把 RBM 换成NADE3，NADE Neual Autoregressive Density Estimator 同样为多峰模型建模。Magenta Pianoroll-NADE 实现了这个模型

但是，RBM 和 NADE 的多峰模型较为 noisy，从其生成的序列来看，有一些被sample 出来的点并不很make sense，从这个角度来讲，其效果不如下面要讲的 Events 表示

除了 RBM 和 NADE 之外，Mixture Density Networks 4（用网络来预测GMM的参数）和 RNN 的结合也是一个方向，这个方向似乎还没有人做音乐生成。但是应用在音乐生成上直观来说是一件较为trivial 的事情，并没有特别大的 contribution

其实这种 density estimator 从ML 的角度来说有其研究价值，因为不止音乐这一种数据是时间序列多峰分布的。如果想要从这方面寻求突破的话，就需要寻找更好的 density estimation，这要求熟悉 ML 相关背景

 

Events 表示

因为midi本身就是一个 events 序列，所以可以把events做encoding。但midi events 包含连续变量（按键力度，时间间隔），因此不能直接对 midi events 进行encoding，需要进行离散化处理之后才能做encoding。各个模型的离散化方法各不一样。Magenta Performance RNN 5 就是对 midi events 定义了一组自己的encoding。相当于把多峰分布变成了单峰分布，然后再用softmax 来建模。我将这组encoding 简化之后训了一个LSTM 模型，可以work，但较难 evaluate。

 

加入乐理知识的混合表示

各个琴键 notes 是可以被分类到各自的tone，也可以被分内到 Scale；和弦 Chord 是有一定规则的；按键组合 Progression 也需要依据一定的规则。这些规则可以在encoding 的时候被定义。Magenta Lookback RNN6 就是一个例子。同时，这些规则还可以被当做 RL 的reward，RL Tuner 7 就做了这件事情。RL的文章一开始想投ICLR，但因为 evaluation 被reviewer 质疑，降到了workshop 接受。后来RL 文章换了一个讲故事的方法（模型完全没变），补充了一个实验，发到了 NIPS 上。

 

对于midi velocity 的处理

midi 的编码不仅encode 了哪个键，也encode 了要用多大的力气按下这个键。这在 Performance RNN 中以 events 的形式来体现，这个events 被定义为力度变换，当出现了这个event 之后，以后按键的力度就根据这个event 的信息来按。相比之下，midi 的原始encoding 中，力度和哪个键是被encode 到同一个event中，而performance RNN 把力度和键拆开了。

在 Song From PI8 中建立了一个边生成边标注的模型，上一层生成音符，下一层对生成的音符标注velocity，这样相当于用另一种方法做了跟 Performance RNN 同样的事。

 

对于 midi multi-track 的处理

Song From PI 不止在标注，同时还在根据生成的音乐去接着生成和弦和鼓点。这是一件multi-track 的事情，也似乎是第一个做multi track的文章。当然这个工作把鼓点的生成也作为了一个标注任务。所以这个工作最终是同时生成了音乐，鼓点，和弦。但是从给的demo 来看，鼓点很trivial，就只是在每个timestep 敲一下。

应该说明的是这份工作比Magenta 的工作其实做得更多，确实很不容易。但是因为其论文中 evaluation 只有人类打分，没有更加客观的评价指标，在ICLR中被reviewer 攻击，降级到了 workshop track。

 

五线谱符号表示

除了midi 之外，另一个表示是五线谱，这种表示就完全是把这个任务当做一个NLP 任务来做了。DeepBach9 是少有的在音乐生成上成功发了论文的工作，他们的文章发在了ICML 17. 除了encode 了很多音乐 music theory 的信息之外，他们还做了一个人和机器协同作曲的demo，我觉得这个demo 是他们的工作可以被接受的一个很大原因。与此同时，Magenta 和 song from PI 的工作都有很多的 music theory encoding，但并没有得到特别的认可。

 

学术贡献的方向和难点

Magenta RL 一开始用music theory 来做reward，实际上我觉得这个工作很solid，但是在 ICLR 的reviewer 质疑了他们的evaluation太过主观，因此被降级到了 workshop。后来他们完全没有改模型，但是从ML的角度重新讲了这个故事，就被NIPS 接受了。Song from PI 相比Magenta 的提高其实是不错的，因为他们首次建模了multi track。我之前的周报中也写到了边生成边标注这个想法。但他们的evaluation 只是让人打分，也被ICLR reviwer 质疑，也降级到了workshop。另外的很多相关工作最后发表的venue 都是workshop，都被质疑Evaluation。在Bengio 12年的RNN-RBM 之后，只有DeepBach 发了ICML。DeepBach 的一个很重的点就是让人和机器协同作曲，我觉得这是他们的文章可以被接受的一个很重要的原因

生成模型的Evaluation 一直是一件比较难的事情。音乐生成现在的community 是没有一个标准的 evaluation 方法。BLEU 没有 reference，perplexity 也没有被提出过。

音乐数据本身其实是一个比较有意思的数据。如果我们讨论 piano roll 这种representation，首先它是一个 time series，然后在每个timestep 上它是一个多峰分布。键和键之间由于和弦是有规则的，所以 corelation 很强。把一个midi 时间序列画出来之后，可以观察到它是很多个峰在时间轴上周期出现。在中期内，一个峰相当于一个句子，在长期内，类似的句子不断重复，形成周期性的pattern，因为每个句子都类似，所以数据具有 global coherence，而句子与句子之间又有不同，所以具有 local variance

Global coherence 和 Local variance 是有trade off 行为的。具体来说，如果一个序列 globally 显示出很强的周期性，那么这个序列就相当于有比较好的 global coherence，因而locally 的变化就不多；相对应地，如果一个序列locally 变化多，那么globally 周期性就比较差，coherence 就比较小。

因此衡量这种tradeoff 是我能想到的一个evaluation。为了衡量 global coherence，我们需要去 measure 一个序列到底 有多么周期 / 多像一个周期序列。比如说，我想要这么一个函数，这个函数当接收一个正弦波的时候输入1和pi，表示这是一个周期为pi序列，当接收一个random noise 的时候输出0和inf，表示这是一个非周期序列，周期为infinity。但我不知道数学上有没有这样子的函数看，似乎Auto-corelation 可以做这个事情，不过这个还需要进一步验证。

然后，衡量 local variance 的方法。首先需要定义有多么local。如果以单个 timestep 作为时间粒度，然后用前后timestep 只差作为local variance，确实可以体现出来一些东西，但是这个measure 非常不 reasonable。

另一个衡量local variance 的思路是，给定timestep的数量，比如说10个，然后对长度为10的序列做DTW Dyna mic time wrapping 10。这种DTW 可以衡量两个序列的相似程度。但它要求两个序列差不多长。如果要做DTW，需要对序列做切割，但是如何切割又是一个问题。

需要注意的是，这种Global coherence 和 Local variance 在语言中也是很常见的。虽然没有measurement，但语言上的local variance 不是一个问题，global coherence 才是一个问题。如何衡量生成的一个句子前后说的是同一件事情，而不会drift 到不同的topic，这也 remain to be investigated
