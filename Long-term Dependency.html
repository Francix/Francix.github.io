Long-term Dependency and Global Coherence in RNNs

 

符尧 francis_yao@pku.edu.cn

北大信科

2018年3月20日

 

音乐具有long term dependency

音乐数据的一个显著特征就是其 time steps 非常长，如果我们用Magenta Performance RNN 1 的表示方法（Events），那么一段15秒的音乐在quantize 之后其sequence的长度为 300 左右，相比之下这算是一个比较长的序列

  • 相比是指，一个RNN 语言生成模型一般生成到20个 time step 就算比较长的了，PTB language model 在 tf tutorial 的长度是200，而 checklist 的长度是 60（Summarization 和 MT 长度暂时不知道）

那么，在这个problem setting 中，确实音乐存在long term dependency

同时，如果用piano roll representation 2，一段训练集的长度也是在200-400 之间，也是一个比较长的序列，所以总的来说音乐是一个比较具有long-term dependency 的数据

 

Long term dependency 和 Global Coherence 的含义差别

Long term dependency 更加偏向于局部呼应，这是说，假设一个句子有 100 个词，第 1 个词和第 100 个词组成前后呼应的关系，这个叫做 long term dependency。同时这个dependency 应该是局部的，第1个词和第100个词紧密相关，但第2个词可以和第 100个词毫无关系

一个典型的例子是 linux source code 3中，前后括号的呼应。有了前括号就必须要有后括号，当前括号和后括号之间的内容很多时，模型依然记得加上后括号，这叫做模型有 long term dependency。但前后括号中间是什么内容，模型不用管

Global coherence 是说，一个很长的句子的每一个部分都必须是描述某个整体的一个方面，局部和局部之间要有连续的变化 (monotonic change)，一个例子是：举头望明月，低头思故乡，前后两句连续变化，中间的内容和两端的内容不可分开

当研究者们在谈论RNN 的建模能力的时候，long-term dependency 和 Global coherence 常常是混用的。这两点都是长序列的性质，但是其实是两个有差别的方面

 

缺乏衡量long-term dependency 和 Global Coherence 的工具

即使我们知道音乐有这么一个 long term dependency 性质，但是我们也没有一个好的metrics 去衡量说，我生成的内容确实是有这么一个 global coherence

具体来说，我无法比较我生成的两个句子，句子1比句子2 有更好的 long term dependency

我也无法比较我生成的两个句子，句子1 比句子2 有更好的global coherence

因为我们缺乏这两种工具，所有我们在做研究的时候，非常被动。在之前对RNN的研究中，更多是从优化的方面来看long-term dependency，认为把gradient 给稳定了之后，就可以学习到long-term dependency了。确实这样做可以取得很好的效果，但是现在的状态是，LSTM可以在长序列中收敛，在分类模型和表示学习中效果惊人，但在生成模型中，非常不稳定，Long term dependency 比较好，Global coherence 很差。到现在这个研究阶段，我们更多希望从 semantics 这一级别来measure long-term dependency，而不是从优化这一级别

因而我们需要好的衡量 long term 的工具，需要好的 baseline 任务

 

分类任务不是衡量 long-term dependency 的好任务

因为long term dependency 更多依赖于intra-sequence structure，但分类任务更多依赖于surface features

一个很明显的例子是，对于一个序列分类任务，当我的 representation learning 做得足够好的时候，对 sequence 的每个 timestep 做pooling 就已经能有有比较好的效果了，但pooling 从数学上看明显无法衡量一个序列的 intra-sequence relations。当我们做 pooling的时候，第一个词和第一百个词没有任何区别

因此要找到一个好的 long-term dependency 的任务，需要一个对句子内部结构有要求的任务

 

没有Condition 的 Language Model 不是衡量 long-term dependency 的好任务

因为有人使用暴力地使用 8-gram language model 发现这样做可以有非常好的 perplexity，但是 8-gram model 明显不是 long-term

同时，Sebastian Riedel 的研究也发现 ptb language model的时候，self-attention只要对前五个 timestep 做attention就已经可以得到足够好的 perplexity

因此，language model 虽然是一个非常重要的NLP Baseline Tasks，但是它确实对 long-term dependency 要求不高

 

但是有一些模型确实对 longer term 表现得更好

我自己的 Natural QA 中，baseline seq2seq 比较倾向于生成短序列，而增加了 attention 作为 decoder language model 的 condition 之后，模型更加倾向于生成长句子

在 Checklist 这个模型中也 report 了当增加一个 memory 作为 decoder language model 的时候，模型在生成长句子的时候表现得更好

再同时， attention， additional loss (regularization)，reinforment tuning 都是增加 long-term dependency 的有效手段

 

音乐的特征是 Local-variance, Global-coherence

回想乐理知识：音乐创作的终极目标是生成打动人心的主旋律。在有了主旋律之后，所有的句子就是对主旋律的变化 + 重复

比如说一段流行音乐，其句子与句子之间是有明显变化的，但这些句子是从同一个主旋律之中变化而来的。

如何在生成模型中，生成具有local variance - global coherence 的句子是一个值得探究的问题

同时，很多NLP 的数据也具有 local variance global coherence 的强烈特征，一个典型的例子是诗歌，虽然现在大多数诗歌生成的模型都不怎么样：只能写景，不能抒情。只能简单把一个给定意境中的景物并列，无法从意境中提炼情感

但是我们依然缺乏比较好的手段去对 local variance 来modeling 和 measuring. 我希望找到一种 metrics，它可以告诉我两个句子的用词有多相似，即使它们不属于同一主题；然后有另一种 metrics，它可以告诉我两个句子是否属于同一主题，即使它们的用词不一定相似
